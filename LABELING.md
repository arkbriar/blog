# Post Labeling Guide

## Categories

Each post has exactly one category. Title case, English.

| Category | Scope |
|---|---|
| **Algorithm** | Sorting, searching, DP, string algorithms, general algorithmic techniques |
| **Data Structure** | Trees, heaps, hash tables, filters, queues |
| **Graph Theory** | Matching, min-cut, network flow, bipartite graphs |
| **Mathematics** | Number theory, combinatorics, game theory |
| **Systems** | Networking, databases, distributed systems, middleware |
| **Programming** | Language features, design patterns, tools, JVM |

When a post spans multiple areas (e.g., a contest problem involving number theory), pick the category that best describes the core technique, not the motivation.

## Tags

- 2-4 tags per post
- Lowercase, kebab-case, singular form, English
- Describe specific techniques or topics (e.g., `segment-tree`, `dynamic-programming`)
- Do not duplicate the category name as a tag (no `algorithm` tag on an Algorithm post)
- Platform tags (`hackerrank`, `leetcode`, `atcoder`, `project-euler`) are fine when the post is a contest/problem writeup

## Content Formatting

- Always insert a space between adjacent CJK and Latin/number characters
- No space before CJK punctuation (。，；：！？、)
